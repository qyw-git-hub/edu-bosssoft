    # 前端规范

    # 页面规范

    • 页面内容保持一屏幕显示完整内容，避免出现滚动条（w-panel）
    • 所有界面风格统一（如按钮、表格、表单、提示语、弹窗等）根据使用组件即可。
    • 间距紧凑，不同区域内容需要分割线
    • 交互小手（可点击的需要加小手图标）

    # 表格规范

    • 表头标题全部居中，金额默认靠右，能预估宽度的(例如姓名)，就固定宽(不设置最小宽)居中显示，不确定字符长度的(例如描述、备注)，居左显示。
    • 金额表头必须加单位(元)，金额的数据保留2位小数(按照实际情况保留)，若处理了千分位，涉及到导出与后端约定一起修改。
    • 涉及到筛选表头，视情况调整列宽。
    • 页脚合计，根据情况是否加单位，需注意tooltip提示内容是否和数据保持一致(具体情况根据产品定)。

    # 弹窗规范

    • 弹窗避免出现双重滚动条
    • 弹窗出现的查询条件，默认展示3个，大弹窗展示4个。

    # form表单

    • 所有弹窗内的 form 表单中取消冒号(😃，统一右对齐，查看弹窗使用的description标签也默认取消冒号(根据设计来)。

    # 搜索条件

    • 搜素条件默认一排4个查询条件，展示2排。若左侧有slot占位，那么右边的查询条件默认展示3个
